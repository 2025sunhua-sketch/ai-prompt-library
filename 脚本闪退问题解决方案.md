# 批处理脚本闪退/乱码问题解决方案

## 问题原因

你遇到的错误信息：
```
'夎' 不是内部或外部命令
'it' 不是内部或外部命令
'o' 不是内部或外部命令
璇疯緭鍏？SSH 浠撳簱鍦板潃...
```

**根本原因**：这是**文件编码问题**。

- Windows 命令行默认使用 **GBK 编码**（代码页 936）
- 但脚本文件是 **UTF-8 编码**
- 导致中文字符被错误解析成乱码和无效命令

### 方案 B：使用 PowerShell 执行中文版 ⭐⭐⭐⭐

如果你更习惯中文界面，可以使用 PowerShell 运行：

**步骤**：

1. 在项目文件夹空白处，按住 `Shift` + 右键
2. 选择"在此处打开 PowerShell 窗口"
3. 执行以下命令设置 UTF-8 编码：
   ```powershell
   [Console]::OutputEncoding = [System.Text.Encoding]::UTF8
   ```
4. 然后运行脚本：
   ```powershell
   .\init-git-simple.bat
   ```

### 方案 C：手动转换编码 ⭐⭐⭐（技术向）

如果你想修复原有的中文脚本：

**方法 1：使用记事本**
1. 用记事本打开 `init-git.bat`
2. 点击"文件" → "另存为"
3. 在"编码"下拉框选择 **"ANSI"**
4. 保存覆盖原文件

**方法 2：使用 VBScript 自动转换**
```cmd
cscript convert-to-gbk.vbs init-git.bat init-git-gbk.bat
```
然后运行生成的 `init-git-gbk.bat`

## 为什么会出现编码问题？

### 技术解释

Windows 命令行（CMD）的默认编码：
- **简体中文系统**：GBK（代码页 936）
- **英文系统**：CP437 或 CP850

而现代编辑器（VSCode、Notepad++ 等）默认保存为 **UTF-8** 编码。

当 UTF-8 编码的中文在 GBK 环境下显示时：
- UTF-8 的中文字节被 GBK 解码器错误解析
- 产生乱码字符（如 `夎`、` 璇疯緭鍏？`）
- 这些乱码被当作命令执行，自然报错

### 为什么英文版没问题？

英文字符（ASCII）在 UTF-8 和 GBK 中的编码是**完全相同**的：
- 字母 A-Z, a-z：0x41-0x5A, 0x61-0x7A
- 数字 0-9：0x30-0x39
- 常见符号：完全一致

所以纯英文脚本在任何编码环境下都能正确显示和执行。

## 常见问题 Q&A

### Q1: 我可以用中文脚本吗？

**可以**，但需要确保：
1. 文件编码是 ANSI（GBK）
2. 或者在 PowerShell 中设置 UTF-8 输出编码

**推荐**：直接用英文版，更省心。

---

### Q2: 输入的内容可以是中文吗？

**可以**！虽然脚本提示是英文，但你输入的用户名、邮箱等信息支持中文：
- Username: 必须是英文（GitHub 账号名）
- Email: 标准邮箱格式
- Repository URL: 必须是英文（Git 仓库地址）

---

### Q3: 如果还是闪退怎么办？

检查以下几点：
1. **是否以管理员身份运行**：右键 → "以管理员身份运行"
2. **是否有 Git**：在 CMD 中输入 `git --version` 确认
3. **是否在正确的目录**：确保在 `ai-prompt-library` 文件夹内运行

如果还有问题，使用**手动方式**：
```cmd
git init
git config user.name "YourName"
git config user.email "your@email.com"
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YourName/ai-prompt-library.git
git push -u origin main
```
