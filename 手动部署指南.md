## 🌐 第三步：创建 GitHub 远程仓库

### 3.1 登录 GitHub

访问 https://github.com/ 并登录你的账号。

**如果没有账号**：点击 "Sign up" 注册（需要邮箱验证）。

### 3.2 创建新仓库

1. 点击右上角 **+** → 选择 **"New repository"**
2. 填写仓库信息：
   - **Repository name**: `ai-prompt-library`（必须和你的项目名一致）
   - **Description**: `AI 提示词优化与模板库 - 免费工具导航`
   - **Visibility**: 选择 **Public**（公开，GitHub Pages 免费版要求）
   - **Initialize this repository with**: ❌ **不要勾选**（保持空仓库）
3. 点击 **"Create repository"**

### 3.3 复制仓库地址

创建成功后，页面会显示仓库地址，有两种格式：

**HTTPS 方式**（推荐新手）：
```
https://github.com/你的用户名/ai-prompt-library.git
```

**SSH 方式**（需要配置 SSH 密钥）：
```
git@github.com:你的用户名/ai-prompt-library.git
```

**复制 HTTPS 地址备用**。

## 🔗 第四步：关联远程仓库并推送

### 4.1 回到 Git Bash

确保你还在项目的 Git Bash 窗口中。

### 4.2 添加远程仓库

```bash
# 将远程仓库命名为 origin（标准命名）
git remote add origin https://github.com/你的用户名/ai-prompt-library.git

# 验证是否成功
git remote -v
```

**预期输出：**
```
origin  https://github.com/你的用户名/ai-prompt-library.git (fetch)
origin  https://github.com/你的用户名/ai-prompt-library.git (push)
```

### 4.3 推送到 GitHub

```bash
# 推送到 main 分支（GitHub 默认分支名）
git branch -M main
git push -u origin main
```

**⚠️ 身份验证问题：**

从 2021 年起，GitHub 不再支持账号密码推送，你需要：

#### 方案 A：使用 Personal Access Token（HTTPS 方式）

当 Git 提示输入密码时：
- **Username**: 你的 GitHub 用户名
- **Password**: **不要输入真实密码**，而是输入 Personal Access Token

**生成 Token 的步骤：**
1. 访问 https://github.com/settings/tokens
2. 点击 **"Generate new token (classic)"**
3. 填写：
   - **Note**: `ai-prompt-library-deploy`
   - **Expiration**: `90 days`（或更长）
   - **Select scopes**: ✅ 勾选 `repo`（全部权限）
4. 点击 **"Generate token"**
5. **立即复制生成的 token**（只显示一次！）

下次推送时，用这个 token 代替密码。

#### 方案 B：配置 SSH 密钥（推荐长期使用）

```bash
# 1. 生成 SSH 密钥（一路回车即可）
ssh-keygen -t ed25519 -C "你的邮箱@example.com"

# 2. 查看公钥内容
cat ~/.ssh/id_ed25519.pub

# 3. 复制输出的内容（以 ssh-ed25519 开头）
# 4. 访问 https://github.com/settings/keys
# 5. 点击 "New SSH key" → 粘贴公钥 → 保存

# 6. 测试连接
ssh -T git@github.com
```

看到 `Hi 你的用户名! You've successfully authenticated...` 即成功。

## ⚙️ 第五步：启用 GitHub Pages

### 5.1 进入仓库设置

1. 访问你的仓库页面：`https://github.com/你的用户名/ai-prompt-library`
2. 点击顶部菜单的 **"Settings"**

### 5.2 配置 Pages

1. 左侧菜单找到 **"Pages"**（可能在 "Code and automation" 分类下）
2. **Build and deployment**:
   - **Source**: Deploy from a branch
   - **Branch**: 选择 `main` / `root`
3. 点击 **"Save"**

### 5.3 等待构建

GitHub 会自动开始构建，大约需要 **2-5 分钟**。

你可以：
- 刷新页面查看构建进度
- 或点击 **"Actions"** 标签页查看构建日志

## 🛠️ 常见问题解决

### Q1: 推送时提示 "Permission denied"

**原因**：身份验证失败

**解决**：
- HTTPS 方式：确保使用 Personal Access Token 而非密码
- SSH 方式：重新配置 SSH 密钥

### Q2: 国内访问速度慢或无法访问

**原因**：GitHub 服务器在国外

**解决方案**：
1. **方案 A**：配合 Cloudflare CDN 使用（见下方进阶教程）
2. **方案 B**：使用 GitHub 加速服务（如 FastGit）
3. **方案 C**：继续用 Cloudflare Pages Direct Upload

### Q3: 页面显示但无样式

**可能原因**：
- CSS 文件路径错误
- 文件编码问题（确保 UTF-8）
- 缓存问题（强制刷新 Ctrl+F5）

**检查方法**：
1. 在浏览器按 F12 打开开发者工具
2. 查看 Console 是否有报错
3. 查看 Network 中 CSS 文件是否成功加载（状态码 200）

### Q4: 想更换自定义域名

**步骤**：
1. 购买域名（如 namecheap、GoDaddy）
2. 在仓库 Settings → Pages → Custom domain 填写域名
3. 在域名服务商处配置 CNAME 记录指向 `你的用户名.github.io`
4. 等待 DNS 生效（几分钟到几小时）

## 📊 GitHub Pages vs Cloudflare Pages 对比

| 维度 | GitHub Pages | Cloudflare Pages |
|------|-------------|-----------------|
| **部署方式** | Git 推送 | Direct Upload / Git |
| **构建时间** | 2-5 分钟 | 即时（Direct Upload） |
| **国内访问** | 不稳定 | 较稳定 |
| **学习成本** | 需学 Git | 低 |
| **博客功能** | 强（配合 Hugo/Jekyll） | 弱 |
| **适合场景** | 技术博客、文档站 | 纯前端应用、工具站 |
